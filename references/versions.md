# UI Forge — Version Compatibility Matrix

UI Forge is a zero-dependency Node.js skill. This matrix lists the minimum
versions of hosts, framework targets, companion skills, and component libraries
UI Forge has been validated against.

Absence of a row means "not yet tested" — it does not imply incompatibility.

---

## Runtime

| Component | Minimum | Notes |
|---|---|---|
| Node.js | 18.0.0 | ESM, no transpile. 20.x recommended. |
| npm | 9.0.0 | For `npx skills add` installation. |

## AI host

| Host | Minimum | Notes |
|---|---|---|
| Claude Code | 1.0 | Primary target — Skill tool + vision + file tools. |
| Cursor / Cline / Copilot | — | Usable; no first-class Skill integration. |
| Anthropic SDK-based agents | claude-sonnet-4-6 or newer | Vision-capable models only when `+IMAGE` is used. |

## Framework targets

| Target | Minimum | Notes |
|---|---|---|
| Next.js | 14.x (Pages Router or App Router) | Tested on 14, 15. |
| React | 18.2 | Hooks + concurrent features assumed. |
| TypeScript | 5.0 | Interface parsing assumes modern syntax. |
| Tailwind CSS | 3.3 | Earlier versions work; theme extraction may be less complete. |

## Companion skill — StackShift

| StackShift | UI Forge contract mode | Notes |
|---|---|---|
| < 0.1.5 | not supported | Variant Router protocol required. |
| 0.1.5 – 0.1.x | `CONVERT_VARIANT` v1.0.0 | Current handoff contract. |
| 0.2.x (planned) | `CONVERT_VARIANT` v1.x | Backward compatible; contract version tag preserves old behavior. |

The handoff contract is versioned via the `/** @contract-version x.y.z */`
JSDoc tag on the exported props interface. Absence = `1.0.0`.

## Supported contract versions (UI Forge 0.1.3)

| Version | Status |
|---|---|
| 1.0.0 | supported (current) |

Unknown versions are still processed — UI Forge emits a stderr warning and the
generated FORGE NOTES flag the unrecognised version. Callers should upgrade
UI Forge when their StackShift version emits a newer contract tag.

## Component libraries

UI Forge does not depend on any library — it reads your project's
`design-arch.json` and swaps to whatever you use. The table lists libraries
that have been validated in conversion examples.

| Library | Version tested | Notes |
|---|---|---|
| shadcn/ui | current (any 2024–2026 revision) | Primary target. |
| Material UI (MUI) | 5.x | Component swap tested for Button, Card, TextField. |
| Mantine | 7.x | Limited validation. |
| Chakra UI | 2.x | Limited validation. |
| Radix UI primitives | 1.x | Usually proxied via shadcn. |
| Headless UI | 2.x | Usually proxied via shadcn. |

## Styling systems

| System | Support |
|---|---|
| Tailwind utility classes | First-class — extracted and token-mapped. |
| CSS Modules | Classnames extracted; values not extracted (semantic hints only). |
| CSS-in-JS (styled-components, emotion) | Template literal CSS extracted. |
| Vanilla CSS (`<style>`) | Extracted from HTML refs. |
| CSS variables / custom properties | Read from `globals.css` excerpt. |

## Node module marker files

UI Forge looks for the following marker files to enable mode-specific behavior:

| Marker | Triggers |
|---|---|
| `design/design-arch.json` | Required. Generated by `scripts/scan.js`. |
| `design/forge-page-plan.json` | Stage 2 of the two-stage page pipeline. |
| `.stackshift/installed.json` | Paired-mode detection. Surfaces in output; honors `a11yRequired`. |

## Breaking change policy

- **Patch (x.y.Z)**: bug fixes, new signals that default off, new flags that default off.
- **Minor (x.Y.z)**: new required behavior on opt-in modifiers; contract version can advance.
- **Major (X.y.z)**: schema bumps (design-arch._v), removed signals, changed default output shape.

Each release includes a file in `change-logs/` with the full diff rationale.
