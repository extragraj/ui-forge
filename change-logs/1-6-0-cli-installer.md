# 1.6.0 — First-party install CLI

**Date:** 2026-05-21

## Summary

UI Forge gains a first-party install CLI (`pnpm dlx ui-forge init` / `npx ui-forge init`) that replaces `scripts/cli.js install` and the legacy `npx skills add` install path. One interactive pass wires slash commands, scoped Bash permissions, MCP server registration, optional hooks, and a portable project-root shim. Every change is recorded in a lockfile so `repair`, `update`, `doctor`, and `uninstall` are exact.

## Repository layout

UI Forge becomes a pnpm monorepo:

- **`bin/cli.mjs`** — new CLI entrypoint (delegates to `cli/dist/index.js`).
- **`cli/`** — new TypeScript installer package (`ui-forge-cli`, private to the workspace).
  - `cli/src/{index,install,repair,update,doctor,uninstall,migrate,ls,mcp-config}.ts` — command handlers.
  - `cli/src/{assets,flags,prompts,platforms,pairing,theme,registry,fs-utils,lockfile,legacy-sweep}.ts` — core modules.
  - `cli/src/wiring/{commands,permissions,mcp,hooks,project-cli,forgeignore}.ts` — per-target writers.
  - `cli/dist/` — build output (the only `cli/` content that ships).
- **`scripts/`** — unchanged runtime, still stdlib-only ESM.
- **`pnpm-workspace.yaml`** — declares `cli` as a workspace package.
- `package.json`: `bin` → `bin/cli.mjs`; new `build` / `prepack` scripts; `files` allowlist tightened to publish only runtime + installer.

Runtime dependencies stay zero. Installer dependencies (`@clack/prompts`, plus dev `typescript`) live under `cli/`.

## Command surface

| Command | Purpose |
|---|---|
| `ui-forge init` | Interactive install. Re-run to add/remove features — detects the existing lockfile and uses prior selections as defaults. |
| `ui-forge repair` | Re-apply wiring from `.ui-forge/installed.json` non-interactively. |
| `ui-forge update` | Sync wiring to the current `skill.version`. |
| `ui-forge doctor` | Diagnostics; `--fix` deletes legacy files and prunes stale wiring. |
| `ui-forge uninstall` | Removes everything UI Forge wrote (tracked via lockfile). |
| `ui-forge migrate` | One-shot migration from a pre-1.6.0 install (detects existing slash command files). |
| `ui-forge ls` | One-screen install summary. |
| `ui-forge mcp-config` | Print MCP snippet for manual wiring. |
| `ui-forge help` | Usage. |

Every prompt has a flag equivalent. Full CI example:

```bash
npx ui-forge init --yes \
  --scope=project --platforms=claude,cursor \
  --features=scan,forge,verify --theme=shadcn \
  --pair=auto --mcp=on --mcp-clients=claude-code \
  --hooks=off --project-cli=on
```

## Feature selection

Required (always installed): `scan`, `forge`. Optional: `verify` (bundles `verify-standards.js` + `validate-contract.js` + `packages/variant-contract/`), `export-design`, `fetch-handoff`, `mcp-server`.

The installer copies **only the files each selected feature requires**. A `scan + forge` install drops ~7 files into the target skill dir, not the whole repo. The asset manifest (`cli/src/assets.ts`) is the typed source of truth; `NEVER_COPY` patterns are the safety net (e.g. `cli/src/`, `tests/`, `change-logs/`, `CLAUDE.md`, `scripts/sync-version.mjs`, legacy `scripts/cli.js`).

## Theme + StackShift pairing

- Themes: `shadcn`, `mantine`, `plain-tailwind`, `stackshift`.
- Pairing auto-detected via `.stackshift/installed.json`. When detected, paired-only features (validate hook, `+A11Y` defaults) auto-enable.
- **Stackshift limited mode** (unpaired): `themeOverride` is stripped from the copied theme JSON, paired signals are removed, `_limited: true` is set. Default forgeignore is used instead of the stackshift template. Persisted as `themeLimited: true` in the lockfile.

## Project-root CLI shim (`./ui-forge.mjs`)

Generated at the project root. Resolves `SKILL_ROOT` at **runtime** using the same priority order as `scripts/detect.js` — safe to commit, portable across machines, no hardcoded absolute paths.

```bash
node ui-forge.mjs scan --quick
node ui-forge.mjs forge --task "Convert hero" --refs ./hero.html --output ./Hero.tsx
node ui-forge.mjs verify ./Hero.tsx ./types.ts
node ui-forge.mjs export
node ui-forge.mjs handoff <url>
node ui-forge.mjs mcp
```

Replaces the old `node "$SKILL_ROOT/scripts/cli.js" <cmd>` patterns from the README/SKILL.md.

## Wiring

- **Slash commands**: source-of-truth templates in `commands/*.md` are copied per platform with `$CLAUDE_PLUGIN_ROOT` substituted to the resolved skill root. Each file gets a provenance header `# Generated by ui-forge@<live skill.version>` so `update` produces clean diffs and `uninstall` identifies owned files. New `commands/forge-handoff.md` added for the `fetch-handoff` feature.
- **Permissions**: `settings.json` `permissions.allow` gets one entry per selected script (least-privilege), not the broad `scripts/*` glob. POSIX separators always — Claude Code's matcher works on Windows. Deep merge preserves unrelated entries. `.bak` backup on first patch.
- **MCP**: detects + patches Claude Code, Cursor, Codex, and Cline configs. Cline path resolved per OS (Windows `%APPDATA%`, macOS `~/Library/Application Support`, Linux `~/.config`). Auto-skips clients whose config file doesn't exist.
- **Hooks**: opt-in PostToolUse verify hook; paired StackShift validate hook auto-on under pairing.
- **`.forgeignore`**: written from the appropriate template. Default for most themes; stackshift template only when paired (limited-mode falls back to default).

## Lockfile (`.ui-forge/installed.json`)

```jsonc
{
  "skillVersion": "1.6.0",
  "scope": "project",
  "platforms": ["claude"],
  "paired": false,
  "theme": "shadcn",
  "themeLimited": false,
  "features": ["scan", "forge", "verify"],
  "mcpClients": [],
  "hooks": { "postToolUseVerify": false, "stackshiftValidate": false },
  "projectCli": true,
  "written": ["…files we created…"],
  "patched": [{ "path": ".claude/settings.json", "keys": ["permissions.allow"] }],
  "pruned": []
}
```

- `written[]` — files we wrote (uninstall removes these).
- `patched[]` — JSON/TOML keys we added (uninstall removes only those keys, leaves the file).
- `pruned[]` — legacy-sweep audit trail.

## Legacy sweep

On every `init` (and `doctor --fix`), the installer walks each target skill dir and removes pre-1.6.0 artifacts:

- `examples/`, `tests/`, `change-logs/`, `tmp/`, `CLAUDE.md` (shipped by `npx skills add`).
- `scripts/sync-version.mjs`, `scripts/test-mcp.mjs`, `scripts/cli.js`.
- `cli/src/`, `cli/tsconfig*` (accidental tarball inclusions).
- `themes/*.json` not matching the selected theme.
- `*.bak`, `*.tmp`, `.DS_Store`, `Thumbs.db`.
- Stale slash command files (owned by us per the provenance header but no longer in the selected feature set).
- Stale permission entries pointing into our skill dir.

Unknown files are reported (in `doctor`) or prompted (in interactive `init`); auto-pruned with `--prune-unknown` or `--yes`. Every deletion is logged to `pruned[]`.

## Deprecation + migration

- `scripts/cli.js install` still works in 1.6.0 but prints a deprecation banner pointing at `ui-forge init`.
- `npx skills add extragraj/ui-forge` is removed from README/SKILL.md install docs.
- `ui-forge migrate` detects pre-existing `commands/forge-*.md` files in the new layout, infers the feature set, writes the lockfile, and runs the legacy sweep. Idempotent — re-running on a migrated install is a no-op.
- Planned for **1.7.0**: full removal of `scripts/cli.js` from the repo and tarball.

## Other improvements

- **Atomic writes** — every file goes through `*.tmp` → rename. No half-written `settings.json` on crash.
- **`.bak` backups** — first modification of any JSON/TOML creates a sibling `.bak` (mirrors `scan.js --theme-override`).
- **Cross-platform paths** — `path.posix` for permission strings and command bodies (Claude Code matcher); native separators for filesystem ops. Validated on Windows.
- **Idempotent** — re-running `init` with the same selections produces zero diff. Verified by the test suite.
- **Dry-run** — `--dry-run` on `init`/`repair`/`doctor --fix` prints the plan without touching disk.
- **`sync-version.mjs`** now also bumps `cli/package.json`.

## Tests

`tests/run-cli-tests.mjs` — 59 stdlib-only integration assertions covering:

- `help` output and unknown-command exit.
- `init` filesystem layout (which files are copied, which are NOT), lockfile shape, slash command substitution + provenance header, permission entries (POSIX), project shim with no hardcoded paths, `.forgeignore` template.
- `--dry-run` writes nothing.
- Idempotency (skill dir snapshot + `settings.json` byte-equal across two `init` runs).
- `repair` restores deleted files.
- Legacy sweep on re-install (removes `examples/`, `tests/`, `scripts/cli.js`, `CLAUDE.md`, deselected theme).
- `doctor` exits 0 on a healthy install and reports lockfile presence.
- `ls` summary.
- `uninstall` removes files, settings entries, MCP entries, project shim, and lockfile dir.
- StackShift theme limited mode (unpaired): `themeOverride` stripped, `_limited: true` recorded, `themeLimited: true` in lockfile.
- `migrate` detects pre-existing slash commands, infers features, and is idempotent.

Run with `pnpm test` (or directly `node tests/run-cli-tests.mjs`).
